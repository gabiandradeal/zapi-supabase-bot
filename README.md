# Zapi Supabase Bot

Lê contatos de uma tabela no **Supabase** e envia, via **Z-API**, a mensagem:

```
Olá, <nome_contato> tudo bem com você?
```

Envia para até `MAX_CONTATOS` contatos (padrão: 3).

## Stack

- Python 3.10+
- Supabase (Postgres) — leitura de contatos
- Z-API — envio via WhatsApp

## Setup

**1. Tabela no Supabase** (SQL Editor):

```sql
create table contatos (
    id bigint generated by default as identity primary key,
    nome text not null,
    telefone text not null,
    created_at timestamp with time zone default now()
);

insert into contatos (nome, telefone) values
('João', '5583999999999'),
('Maria', '5583988888888'),
('Pedro', '5583977777777');
```

Telefone no formato `DDI+DDD+número`, sem `+`/espaços (ex: `5583999999999`).

**2. Variáveis de ambiente** — copie `.env.example` para `.env` e preencha:

| Variável            | Origem                                              |
|---------------------|------------------------------------------------------|
| `SUPABASE_URL`      | Project Settings → API                               |
| `SUPABASE_KEY`      | Project Settings → API (publishable/anon key)        |
| `ZAPI_INSTANCE_ID`  | Painel da instância Z-API                            |
| `ZAPI_TOKEN`        | Painel da instância Z-API                            |
| `ZAPI_CLIENT_TOKEN` | Segurança da conta → Security Token (se exigido)      |
| `MAX_CONTATOS`      | Padrão: 3                                            |

> ⚠️ Instâncias novas da Z-API têm **3 dias de trial** e são desativadas
> depois disso. Use credenciais de uma instância ativa.

**3. Instalação e execução**

**No Windows (PowerShell)**
```bash
python -m venv venv
.\venv\Scripts\Activate.ps1
pip install -r requirements.txt
python main.py
```

**No Linux / macOS**
```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python main.py
```

## Notas

- Erros de rede/credenciais são tratados e logados; não interrompem o
  processamento dos demais contatos.
- `.env` nunca deve ser commitado.
